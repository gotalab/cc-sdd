# Steering (Память проекта)

<background_information>
- **Миссия**: Создать или обновить память проекта, чтобы каждая команда могла ссылаться на общие правила и архитектурные ограничения
- **Критерии успеха**:
  - Три основных документа созданы/обновлены
  - Содержимое отражает текущее состояние проекта
  - Паттерны захвачены, а не списки файлов
</background_information>

<instructions>
## Основная задача
Создать или обновить файлы steering на основе анализа кодовой базы.

## Шаги выполнения
1. **Проанализировать кодовую базу**: Структура, технологии, паттерны
2. **Загрузить шаблоны** из `{{KIRO_DIR}}/settings/templates/steering/ru/`
3. **Сгенерировать/обновить** три файла:
   - `{{KIRO_DIR}}/steering/structure.md` - Архитектура, организация, именования
   - `{{KIRO_DIR}}/steering/tech.md` - Технологический стек, фреймворки
   - `{{KIRO_DIR}}/steering/product.md` - Бизнес-контекст, назначение

## Важные ограничения
- Высокоуровневый контент: правила, а не детали
- Захватывать паттерны, не списки файлов
- Сохранять пользовательские правки
</instructions>

## Описание вывода
```
✅ Steering создан/обновлён

## Сгенерировано:
- product.md: {краткое описание}
- tech.md: {технологии}
- structure.md: {паттерны организации}

Проверьте и утвердите как источник истины.
```
